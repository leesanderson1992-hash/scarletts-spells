## Why this changed

The earlier model assumed:
- detected or reviewed misspellings would move directly into the active queue
- the queue itself was the practical source of learning truth

The approved Targeted Writing Practice model now requires:
- a durable writing-issue lifecycle
- child self-correction before formalising a learning gap
- parent final classification
- no Golden Nuggets for checking-only issues
- learning items as the mastery units
- `word_progress` as a compatibility projection only

## What still remains useful here

This file may still be useful as a historical reminder of:
- earlier queue-dedupe expectations
- review cadence expectations
- Gold Bar regression expectations
- the fact that parent review and calm child UX were already important

## What it should not be used for

Do not use this file as the current product contract for:
- issue lifecycle
- Nugget creation
- daily assignment source of truth
- backlog control
- learning-item grouping
