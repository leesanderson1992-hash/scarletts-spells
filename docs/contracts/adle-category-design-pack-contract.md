# ADLE Category Design Pack Contract

## Purpose

This contract makes each 7-UI category design pack part of a repeatable process, not a one-off document.

## Required Sections

Each category design pack must include:

- category scope and taxonomy coverage;
- source artifacts and provenance links;
- category learning primitives;
- reusable template mechanics used by the category;
- micro-skill experience-profile model;
- semantic payload requirements;
- fallback ladder;
- answer-visibility rules;
- accessibility and motion requirements;
- content-readiness requirements;
- schema edge cases;
- rollout groups;
- category completion criteria;
- proof requirements.

## Boundaries

A category design pack may define category-specific learning primitives and interaction language.

It must not redefine:

- evidence semantics;
- scheduler effects;
- reward events;
- persistence ownership;
- global accessibility baseline;
- actual approved teaching copy.

## Standard Category Process

```text
audit content and metadata
-> define category lesson system
-> author representative micro-skill
-> build vertical proof
-> validate owner and child experience
-> refine payloads and primitives
-> freeze category v1 contract
-> complete grouped rollout
```
