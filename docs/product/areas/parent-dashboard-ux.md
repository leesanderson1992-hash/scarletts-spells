# Parent Dashboard UX

## Purpose

This document defines the UX truth for parent operational overview surfaces, especially dashboard-style pages and insight summaries.

## Operational UX rule

Parent overview pages should be:
- compact
- legible
- state-rich
- action-oriented

They are not hero pages.

## Preferred layout pattern

Use:
- tables
- dense row lists
- compact status chips
- inline actions
- one-column reading flow where that reduces split attention

Avoid:
- decorative overview cards that repeat the same signal
- wide two-column workflow layouts on operational pages
- duplicated warning summaries

## Insight truth

Parent summaries should be selector-driven.

That includes:
- recurring totals
- recurring pace
- goal pace
- missed-event summaries
- path summaries

Page-local interpretation should stay minimal.

## Warning behavior

Warnings should be:
- meaningful
- reconcilable to shared data
- few enough to act on

They should not:
- restate the same issue multiple ways
- invent separate truth from course/week surfaces
