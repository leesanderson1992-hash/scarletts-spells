# Life-Skill Project Pedagogy

## Purpose

This document explains why authentic projects and outputs sit at the root of the learning system.

It is not a storage or workflow contract.
It explains the educational method that the contracts must serve.

## Core stance

Children should not meet spelling and writing correction first as detached remediation.

They should meet it inside meaningful work:
- a real task
- a project that matters to them
- an output meant for a real audience or purpose
- a piece of communication worth improving

The system should therefore treat the project as primary and the later correction cycle as responsive support.

## Why authentic work matters

Authentic work does four important jobs:

1. It creates real evidence.
- The child shows what they can currently do without being over-scaffolded by a drill.

2. It creates motivation.
- The child is not only fixing a mistake.
- The child is trying to make something better that already matters.

3. It creates transfer opportunities.
- A skill that appears in real writing tells us more than a skill repeated inside a narrow worksheet pattern.

4. It preserves dignity.
- The child is not defined by an error queue.
- The work stays larger than the correction.

## Key concepts

### Project or task

A project or task is the real piece of work the child is trying to complete.

Examples:
- writing a script
- drafting a message
- composing a description
- answering a lesson question
- preparing notes for a practical outcome

The project is not merely a wrapper around a spelling drill.

### Real-world purpose

Real-world purpose means the work has a function beyond error extraction.

Examples:
- to explain
- to persuade
- to describe
- to plan
- to present
- to document

Purpose matters because it shapes motivation and makes later revision feel coherent rather than punitive.

### Communication output

A communication output is the child-facing thing being produced.

Examples:
- a sentence
- a paragraph
- a script
- a response inside a lesson
- a note or reflection

This is the level at which the system gathers evidence.

### Submission

A submission is the moment the child hands over real work for review.

Pedagogically, a submission means:
- the child has attempted independent performance
- the system now has evidence to inspect
- the parent can respond to real choices, not only practice performance

## Extraction rule

Targeted writing practice must be extracted from meaningful work, not placed in front of it as the default starting point.

The intended order is:

`authentic task -> submission -> parent review -> identified issue -> focused support`

Not:

`worksheet -> correction -> more worksheet -> separate project later`

This does not ban direct practice.
It means direct practice should be a response to evidence, not the whole model.

## Relationship to review and practice

The project layer gives the system:
- context
- evidence
- purpose
- transfer opportunities

The review and practice layers then:
- isolate a specific difficulty
- protect against overreacting to one slip
- offer focused correction or reteaching
- test whether the child can carry the improved skill back into fresh work

That relationship keeps the pedagogy honest.

## Product implication

If the app ever makes practice feel like the main event and authentic work feel like a pretext, the method has drifted.

The product should keep signalling that:
- projects and outputs matter
- reviewed issues are useful evidence
- practice is support
- mastery ultimately means stronger independent performance in real work
